wgrok is an ngrok clone but uses Webex api as a message bus. This is to allow agents, services or anything that is registered to the Webex api to send and receive messages between systems even if there are firewalls or network rules preventing webhooks.

the system is simple. register a bot with developer.webex.com. Use one of the libraries in this folder in your project.

python, go, rust and typescript. There are three components in this repo. 

1. lightweight library for sending a Webex message.
2. Echo bot agent
3. Receiver library. 

The light weight library implements the wgrok spec for sending messages.

v1.0 spec:
message format
Sending: <./echo:<{slug}:payload>>
Receiving: <{slug}:<payload>>
Routing: <senderid>@domain.tld <message> > '^\./echo:' > <senderid>@domain.tld <message>
{slug}: used as a messagebus tag, agent|service|endpoint only acts on the {slug} if it matches

Patterns:

mgrok@webex.bot|mgrok@slack.bot|mgrok@teams.bot > <./echo:{slug}:<payload>> > echo bot > strips ./echo > originator mgrok@webex.bot|mgrok@slack.bot|mgrok@teams.bot <message>

Payload and be anything, otel logs, commands, trace, webhook.

{slug}

.env: sender agent .env will only have sender/receiver token, allowlist and specific slug to match. echo bot .env will token and allow list. 
```
## mgrok sender/receiver
webex_token=<shared token>
target=bot@domain.tld

## mgrok echo
webex_token=<unique token>

## allowlist
domains=domain.tld,*@domain.tld

## slug index optional
{slug}=agentid
```

Implementation - What's in this repo:
* Sending library: in above languages, all this library does is, read webex_token=, wraps input message with ./echo:<message> and sends to .env target= via Webex webhook api.
* echo bot library: in above languages, service that uses https://www.github.com/3rg0n/webex-message-handler for websocket connection to traverse firewalls. Reads .env mgrok echo webex_token=, opens constant websocket to Webex listening for messages, received message is checked against allowlist, removes ./echo: and send output message to originator.
* receiving library: in above languages, message sender checked against allowlist > matches slug to .env > yes, process message, no ignore.
