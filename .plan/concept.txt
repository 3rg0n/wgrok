wgrok — message bus protocol over social messaging platforms

Uses messaging platform APIs (Webex, Slack, Discord) as transport to allow
agents, services, and orchestrators to communicate across firewalls without
inbound webhooks.

== Protocol layers ==

Layer 0 — Direct messaging (base):
  {slug}:{payload}
  Pure pub/sub. Agents listen for their slug. No bot needed.
  Like NATS subjects — agents only act on messages directed at them.

Layer 1 — Verb routing (bot-mediated):
  ./{verb}:{slug}:{payload}
  Messages sent to a bot. The ./ prefix signals "this is a command".
  Bot parses the verb, acts accordingly, delivers {slug}:{payload}.

The ./ convention comes from the original design where a bot handles
context and routing: ./{slug|role} meant "execute this role/action".
In wgrok, the verb IS the role — echo, route, broadcast, store, etc.

== Built-in verbs ==

  echo     — reflect message back to sender (firewall traversal)
  route    — forward to internal system by slug (orchestrator-to-orchestrator)
  broadcast — fan out to all matching subscribers (future)
  store    — persist payload for later retrieval (future)

Only echo is implemented in v1.0. Other verbs are reserved.

== Use cases ==

1. Firewall traversal (echo):
   GitHub Actions orchestrator <-> on-prem orchestrator.
   Both sides share a token. Echo bot relays through Webex.
   ./echo:{slug}:{payload} -> bot -> {slug}:{payload}

2. Multi-agent pub/sub (direct):
   Multiple agents on the same account, each with a unique slug.
   Agents ignore messages not matching their slug. NATS-style.
   {slug}:{payload}

3. Internal routing (route, future):
   Bot receives ./route:{slug}:{payload} and forwards to an
   internal system/endpoint based on slug mapping.

== Components ==

1. Sender library — wraps payload as ./{verb}:{slug}:{payload}, POSTs via REST API
2. Bot — listens on WebSocket, validates allowlist, parses verb, routes accordingly
3. Receiver library — listens on WebSocket, matches slug, delivers to application

== Configuration ==

  # Sender/Receiver
  WGROK_TOKEN=<shared token>
  WGROK_TARGET=bot@domain.tld
  WGROK_SLUG=myagent
  WGROK_DOMAINS=domain.tld

  # Bot
  WGROK_TOKEN=<unique bot token>
  WGROK_DOMAINS=domain.tld,*@trusted.org

  # Optional
  WGROK_DEBUG=true

== Allowlist patterns ==

  domain.tld         -> matches *@domain.tld
  *@domain.tld       -> explicit wildcard
  user@domain.tld    -> exact match

Case-insensitive. Whitespace trimmed. Empty patterns ignored.

== Transport bindings ==

Protocol is transport-agnostic. Requires:
  - REST API for sending messages
  - Persistent connection (WebSocket) for receiving

  Webex:   REST /v1/messages + Mercury WebSocket   [implemented]
  Slack:   chat.postMessage + Socket Mode          [planned]
  Discord: REST /channels/{id}/messages + Gateway  [planned]
